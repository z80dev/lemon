# PR Review Action Plan

Branch: `feat/milestone-m0-m8-implementation`  
Sources: [`claude_review.md`](../claude_review.md), [`codex_review.md`](../codex_review.md)  
Date: 2026-03-16

## Scope

This plan keeps only findings that are either:

- validated directly from the current branch code,
- convertible into a concrete failing automated test or policy/lint check, and
- worth tracking as implementation work rather than style cleanup.

Claude minor/nit items that are purely style, cleanup, or generic coverage suggestions are not carried into this plan unless they map to a concrete defect. Merged duplicates are noted in the appendix.

## Validation Rule

For each retained finding:

- first add the verification test or workflow/policy check,
- confirm it fails on the current branch,
- then implement the fix,
- and treat the item as done only when that verification passes.

## Recommended Order

1. Adaptive/memory correctness issues that can silently disable features or leak scope.
2. Python CLI/WS compatibility issues that make the new client unusable.
3. Skills trust/persistence issues that can corrupt provenance or bypass intended trust posture.
4. Runtime/config/release/CI issues, especially ones that affect production safety or make docs/examples lie.
